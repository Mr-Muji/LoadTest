# <type>(<scope>): <subject>

# 타입 예시:
#   feat      - 새로운 기능 추가
#   fix       - 버그 수정
#   docs      - 문서 변경
#   style     - 코드 포맷, 세미콜론 누락 등
#   refactor  - 코드 리팩토링 (기능 변화 없음)
#   perf      - 성능 개선
#   test      - 테스트 코드 추가/수정
#   chore     - 기타 변경사항 (빌드, 설정 등)
#   ci        - CI 설정 변경
#   build     - 빌드 관련 변경
#
# 스코프 예시: api, frontend, parser, config 등

#   fix(parser): handle nil response
